Skyline

Researchers can test their autonomous and connected vehicles at Mcity's test facility. Infrastructure such as railcrossings and traffic lights can be controlled with software to simulate different driving conditions. Previously, researchers would need to program the infrastructure in person, but the web interface I built this summer enables remote control of the facility using an open source API built by Mcity.

Skyline includes the following features:
• Displays live countdown for when traffic lights will change color
• Only joins Socket IO channel to emit events for infrastructure if main menu panel is expanded
• Shows infrastructure locations and additional information on a map of the test facility
• Allows users to “favorite” infrastructure for ease of access
• Logs all user and API interactions which can be downloaded in a CSV file format
• Main menu automatically scrolls to appropriate location when user clicks on a marker

Using Figma, I collaborated with another intern on the wireframes and prototypes for the initial UI design, which informed our system design decisions later on. I developed the front end using Flask, HTML/CSS, JavaScript, Vue.js, Vuetify, and SQL.

Tags: Front end, UI design, JavaScript, Vue.js, HTML/CSS, Figma

Galactic War Simulation

The program received as input a list of orders that specified whether the trader intended to buy or sell a specific stock. As output, the program would simulate a stock market by printing when a certain stock is bought or sold when it is profitable to do so. I also implemented an algorithm to calculate the ideal time that a time traveler theoretically could have bought shares of a stock and then later sold that stock to maximize profit. Completing this project gave me experience using priority queues and the STL and it helped me become more proficient with object-oriented design.

Tags: C++, Runtime efficiency, stock simulator, STL

Hackapalooza

During my 2017 summer internship at Quicken Loans, there was a month-long hackathon competition called Hackapalooza for interns. Our team chose to solve for the issue identified by the event management team: they relied on delivering contract shuttle drivers packets of paper that held crucial information about team members' transportation arrangements. It was difficult to recreate the packet for each new event and they couldn't contact drivers outside of asking for their personal cell phone numbers.

We created an iOS app called SPOT that automated the process of providing drivers with information for the event. It allowed the event management team to quickly enter information in the app, build a library of common event information, and keep track of their data. Drivers can use the app on a device with cellular and LTE ability so that they can stay in contact with the event management team.

A version of SPOT was implemented by Quicken Loans. Half of my team members had their internships extended to assist with SPOT's implementation.

I was the project manager so I formed and maintained relationships with our stakeholders and contacts, maintained the development schedule, and assisted as needed. I was also responsible for the system design and designed the UI in Sketch in collaboration with other interns.

Tags: Leadership, UI design, Hackathon, Project management, Teamwork

Neurodiversity in Writers

I wrote "Incorporating Universal Design for Learning in the Writing Center" in November 2016. In my paper, I discuss the ableism and disregard that writing tutor guidebooks from the past 20 years have demonstrated toward students with learning disabilities. My research focuses on discussing Universal Design for Learning and practical ways in which undergraduate writing tutors can make their sessions more accessible for a diverse range of students.

I presented my work at the 39th Annual East Central Writing Centers Association Conference in March 2017.
